Computer Networking Project

Hosting a Website with GitHub Pages & Understanding Networking Concepts

Project Overview

This project demonstrates how to set up a website using GitHub Pages while applying key networking concepts like DNS, IP addressing, and protocols.

Steps We Took

  • 1. Set Up a GitHub Repository: Created a public repository for the website.
  • 2. Built the Website: Designed a simple HTML & CSS page.
  • 3. Hosted on GitHub Pages: Enabled GitHub Pages for free website hosting.
  • 4. Configured DNS & IP: GitHub assigns a subdomain and an IP address.
  • 5. Implemented Security: HTTPS is enabled by default for encryption.
  • 6. Verified Networking Aspects: Used `ping` and `traceroute` to test reachability.

Networking Concepts Applied

  • DNS (Domain Name System): GitHub Pages assigns a subdomain (e.g., username.github.io/repository-name). When a user enters the domain, the DNS translates it into an IP address associated with GitHub's hosting servers. The DNS records are managed by GitHub.
  • IP Addressing: GitHub Pages serves the website from a set of IP addresses. We verified this by using the ping command (e.g., ping username.github.io), which returns the public IP address of GitHub’s CDN.
  • HTTP/HTTPS: GitHub Pages enforces HTTPS by default, ensuring secure communication using TLS encryption. This protects against man-in-the-middle attacks and ensures data integrity.
  • Ping & Traceroute: - Ping: Running ping username.github.io verifies that the site is reachable and displays response times.
    - Traceroute: Running traceroute username.github.io (Linux/macOS) or tracert username.github.io (Windows) shows the hops taken to reach GitHub’s servers, demonstrating how data travels across networks.

Challenges & Solutions

Challenge: Setting up GitHub Pages and ensuring the website was accessible globally.

Solution: We enabled GitHub Pages in the repository settings and selected the correct branch for deployment. It took some time for the DNS records to propagate before the site became accessible.

Challenge: Understanding how GitHub assigns IP addresses and how DNS resolution works.

Solution: We used ping and traceroute commands to analyze the network path and verify the assigned IP address.
